Parte 3

Processamento de Vídeo 2023.2

Introdução

A parte 3 do projeto consiste no Desenvolvimento do Sistema de Processamento Visual (SPV), onde será elaborado um algoritmo em python que deverá atender aos requisitos impostos nas partes anteriores. Os códigos serão disponibiizados a seguir.

Objetivos

  • * Aplicar uma quantidade de conceitos de processamento de vídeo da disciplina.
  • * Executar a aplicação com cenas pré-gravadas, para testes e gravação de “demo”.
  • * Executar a aplicação com captura de vídeo em tempo real, mostrando na tela.
  • * Específico: criar algoritmos que realizem o que foi proposto.

Atualização do cenário

Foram definidas pequenas mudanças no funcionamento do projeto. Não serão utilizados componentes físicos, como arduíno, para a aplicação do projeto: os alarmes serão sonoros (disparados pelo próprio computador) e visuais (notificação de alerta no computador / outro dispositivo). Além disso, será realizada uma tentativa de utilizar o horário do sistema para automatizar o processo, principalmente no que se refere aos períodos noturno e diurno definidos anteriormente, com a possibilidade de desativar os filtros através de uma tecla.

Desenvolvimento

Algoritmos

Inicialmente, como solicitado pelo professor, foi criado um algoritmo que utiliza vídeos pré-gravados como teste de análise inicial. O programa a seguir, em tese, realiza a leitura de um vídeo e devolve duas janelas como resultado, um contendo o vídeo original e outra contendo o sistema de filtros, a depender do horário do sistema. Além disso, ambas as saídas detectam movimento e emitem um sinal sonoro caso o movimento seja detectado, além de enviar uma mensagem para o e-mail do usuário através de um e-mail do google (no caso, foi utilizada uma conta do google criada para o propósito desse projeto, e não uma conta pessoal). O sistema de filtros se trata de uma equalização para o período noturno e filtro bilateral para o período diurno. Os aquivos utilizados estão disponíveis AQUI. O arquivo referente a esta etapa está nomeado como "por_video.py". Além disso, é necessário baixar também os arquivos de vídeo, som (em Ogg) e de email.py na mesma pasta do programa principal.
